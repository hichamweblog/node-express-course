## рҹҺҜ Hook: Why E-Commerce Is the Ultimate Backend Challenge

Every second, Shopify processes **$10,000+ in transactions**. Amazon handles **400+ orders per second** during Prime Day. Behind these numbers lies one of the most complex backend architectures in software engineering.

E-commerce isn't just CRUD. It's **CRUD with money on the line**.

### Why E-Commerce Is Architecturally Different

Most web apps are "fail-safe" вҖ” if something goes wrong, you retry or show an error. E-commerce is **"fail-dangerous"** вҖ” mistakes cost real money.

| Normal Web App | E-Commerce App |
|---------------|----------------|
| Duplicate request вҶ’ minor annoyance | Duplicate request вҶ’ **double charge** |
| Stale data вҶ’ slightly wrong display | Stale data вҶ’ **overselling inventory** |
| Lost request вҶ’ user retries | Lost request вҶ’ **lost revenue** |
| Slow response вҶ’ user waits | Slow response вҶ’ **cart abandonment** |
| Bug in logic вҶ’ feature doesn't work | Bug in logic вҶ’ **financial liability** |

This means e-commerce backends need:

1. **Idempotency** вҖ” Processing the same request twice must produce the same result
2. **Transactions** вҖ” Multi-step operations must succeed or fail as a unit
3. **Eventual consistency** вҖ” Some data can be slightly stale (product views), other data must be exact (inventory)
4. **Audit trails** вҖ” Every financial operation must be traceable

## рҹ’Ў Pro Tips vs Junior Traps

| Aspect | рҹҹў Pro Tip | рҹ”ҙ Junior Trap |
|--------|-----------|----------------|
| **Money handling** | Store prices as integers (cents): `$19.99 вҶ’ 1999`. Use Decimal/BigInt for calculations. Never use floating point for money. | Using `float` for prices: `0.1 + 0.2 = 0.30000000000000004` вҖ” customer gets charged wrong amount |
| **Inventory** | Reserve stock during checkout, release if payment fails. Use database transactions. | Deducting stock when added to cart вҖ” items get "locked" by abandoned carts forever |
| **Idempotency** | Use idempotency keys for payment operations. Same key = same result, no matter how many times called. | No idempotency вҖ” webhook retry charges customer twice |
| **Cart design** | Support both guest carts (session-based) and user carts. Merge on login. | Requiring login before cart вҖ” 35% of customers abandon carts when forced to register |
